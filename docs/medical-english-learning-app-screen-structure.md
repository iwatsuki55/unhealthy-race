# 医療問診向け英語学習アプリ 画面構成

## 1. 画面設計の方針

最初のMVPでは、短時間で学習を始められることを優先する。そのため、ホームからすぐに「話す」「読む」「ロールプレイ」「復習」に入れる構成にする。

同時に、将来的に問診支援アプリへ発展できるよう、コンテンツの軸を「症状」「診療場面」「問診目的」で整理しやすい情報設計にしておく。

## 2. 全体マップ

### MVPで必須の画面

1. オンボーディング
2. ログイン / アカウント作成
3. ホーム
4. 学習カテゴリ一覧
5. レッスン詳細
6. スピーキング練習
7. リーディング練習
8. AIロールプレイ
9. 結果 / フィードバック
10. 復習一覧
11. プロフィール / 学習設定

### 将来追加しやすい画面

1. 診療科別シナリオ一覧
2. 症状別フレーズ検索
3. 現場用クイック問診補助
4. 法人管理画面
5. 進捗分析ダッシュボード

## 3. 各画面の役割

### 1. オンボーディング

目的:
アプリの価値を短く伝え、最初の学習テーマ選択につなげる。

主な要素:
- 問診英語特化であることの説明
- できることの紹介
- 利用対象の確認
- 初回おすすめコースへの導線

### 2. ログイン / アカウント作成

目的:
進捗保存、復習、自分向け最適化のためにアカウントを作る。

主な要素:
- メールログイン
- ソーシャルログイン
- 利用規約、注意事項

### 3. ホーム

目的:
今日何を学ぶかを最短で決められるようにする。アプリ全体の起点になる最重要画面。

主な要素:
- 今日のおすすめレッスン
- 直近の苦手項目
- 「話す」「読む」「ロールプレイ」「復習」へのショートカット
- 進捗サマリー
- 継続日数や達成数

将来拡張:
- 現場直前モード
- よく使う症状別ショートカット

### 4. 学習カテゴリ一覧

目的:
症状や目的から練習内容を選べるようにする。

主な要素:
- 症状別カテゴリ
- 問診の目的別カテゴリ
- 難易度フィルタ
- 診療場面フィルタ

想定カテゴリ例:
- 発熱
- 咳
- 腹痛
- 痛み評価
- 既往歴確認
- 薬剤確認
- アレルギー確認

### 5. レッスン詳細

目的:
そのレッスンで何を学ぶかを明確にし、学習モードを選ばせる。

主な要素:
- 学習テーマ
- 到達目標
- キーフレーズ
- 学習モード選択
  - スピーキング
  - リーディング
  - ロールプレイ

### 6. スピーキング練習

目的:
問診フレーズを実際に声に出して練習する。

主な要素:
- 日本語または状況提示
- 話すべき英語のヒント
- 録音ボタン
- 音声認識結果
- 再挑戦導線

評価観点:
- 伝わるか
- 医療現場で自然か
- より安全な表現か

### 7. リーディング練習

目的:
患者の訴えや問診票を読んで、重要情報を拾う力をつける。

主な要素:
- 患者の英文
- 設問
- 選択式または要点抽出
- 解説

評価観点:
- 症状
- 期間
- 程度
- 危険サイン

### 8. AIロールプレイ

目的:
1問1答ではなく、会話の流れとして問診を練習する。

主な要素:
- ケース概要
- 患者AIとの会話UI
- 音声またはテキスト入力
- 会話履歴
- 終了ボタン

評価観点:
- 必要な情報を引き出せたか
- 質問順が自然か
- 表現が分かりやすいか

### 9. 結果 / フィードバック

目的:
学習結果を短時間で振り返り、次の行動につなげる。

主な要素:
- 総合スコア
- 良かった点
- 改善ポイント
- より良い言い換え
- 復習に追加するボタン
- 次のおすすめレッスン

### 10. 復習一覧

目的:
苦手表現や確認漏れを重点的にやり直せるようにする。

主な要素:
- 苦手フレーズ一覧
- 症状別フィルタ
- 間違い理由ラベル
- 再練習ボタン

### 11. プロフィール / 学習設定

目的:
学習体験をユーザー属性に合わせて最適化する。

主な要素:
- 職種
- 英語レベル
- 主な診療場面
- 学習目標
- 通知設定

## 4. 基本導線

### 初回利用フロー

1. オンボーディング
2. アカウント作成
3. 職種とレベル設定
4. おすすめカテゴリ提示
5. 最初のレッスン開始

### 日常利用フロー

1. ホームを開く
2. 今日のおすすめまたは復習を選ぶ
3. 練習を1つ完了する
4. フィードバック確認
5. 必要なら復習登録

### 集中練習フロー

1. 学習カテゴリ一覧
2. 症状やテーマを選択
3. レッスン詳細
4. スピーキングまたはリーディングまたはロールプレイ
5. 結果確認

## 5. ナビゲーション案

スマホアプリ前提なら、下部タブは4つが扱いやすい。

- ホーム
- 学習
- 復習
- 設定

ロールプレイは独立タブにせず、ホームとレッスン詳細の両方から入れる形にする。MVPではタブ数を増やしすぎない方が分かりやすい。

## 6. MVP時点で実装優先度が高い画面

最優先:
- ホーム
- 学習カテゴリ一覧
- レッスン詳細
- スピーキング練習
- リーディング練習
- AIロールプレイ
- 結果 / フィードバック
- 復習一覧

後回しでもよい:
- 細かいプロフィール編集
- 詳細な学習分析
- 法人向け画面

## 7. 将来の問診支援アプリにつながる設計ポイント

- レッスンを「症状」「目的」「診療場面」でタグ管理できるようにする
- フレーズを単独資産として蓄積し、検索しやすくする
- AIフィードバックで「不足質問」を構造化データとして持てるようにする
- ロールプレイ会話を、問診フローのテンプレートとして再利用できる形で管理する

## 8. 次工程への引き継ぎメモ

次のMVP仕様書では、各画面について以下を定義すると実装へ進みやすい。

- 画面ごとの目的
- 入力項目
- 出力項目
- 完了条件
- 最低限必要なデータ構造
- MVPで入れるものと後回しにするもの
